# Community

llmstack is built in the open and we welcome contributions of all kinds -- code, documentation, bug reports, feature ideas, and feedback.

## How to Contribute

1. **Report bugs** -- Found something broken? [Open a bug report](https://github.com/mara-werils/llmstack/issues/new?template=bug_report.yml) with reproduction steps and the output of `llmstack doctor`.

2. **Request features** -- Have an idea? Check the [roadmap](https://github.com/mara-werils/llmstack/blob/main/ROADMAP.md) first, then [open a feature request](https://github.com/mara-werils/llmstack/issues/new?template=feature_request.yml).

3. **Submit pull requests** -- See [CONTRIBUTING.md](https://github.com/mara-werils/llmstack/blob/main/CONTRIBUTING.md) for development setup, code style, and PR guidelines. Keep PRs focused, add tests, and use conventional commit messages (`feat:`, `fix:`, `docs:`).

4. **Write plugins** -- Extend llmstack with new backends or services. See the [plugin interface](https://github.com/mara-werils/llmstack/blob/main/src/llmstack/plugins/spec.py) for details.

5. **Improve documentation** -- Typo fixes, better explanations, new guides -- all welcome. Docs live in the `docs/` directory and are built with mkdocs.

## Where to Get Help

- **GitHub Issues** -- For bugs and feature requests: [github.com/mara-werils/llmstack/issues](https://github.com/mara-werils/llmstack/issues)
- **GitHub Discussions** -- For questions, ideas, and general conversation: [github.com/mara-werils/llmstack/discussions](https://github.com/mara-werils/llmstack/discussions)

## Staying Up to Date

- Star and watch the [GitHub repository](https://github.com/mara-werils/llmstack) for release notifications
- Read the [CHANGELOG](https://github.com/mara-werils/llmstack/blob/main/CHANGELOG.md) for detailed release notes
- Check the [ROADMAP](https://github.com/mara-werils/llmstack/blob/main/ROADMAP.md) to see what's coming next

## Code of Conduct

We are committed to providing a welcoming and respectful environment for everyone. All participants in the llmstack community are expected to:

- **Be respectful** -- Treat everyone with dignity. Disagreements about code are fine; personal attacks are not.
- **Be constructive** -- When giving feedback, focus on the work, not the person. Offer suggestions, not just criticism.
- **Be inclusive** -- Welcome newcomers. Remember that everyone was a beginner once. Avoid jargon when plain language works.
- **Be patient** -- Maintainers and contributors are volunteers. Responses may take time.
- **Act professionally** -- Harassment, discrimination, and abusive behavior will not be tolerated.

If you experience or witness unacceptable behavior, please report it by emailing the maintainers or using GitHub's built-in reporting tools. Reports will be handled confidentially.

## Recognition

Contributors are recognized in pull request threads and release notes. Significant contributions may be highlighted in the README. Every contribution matters, whether it is a one-line typo fix or a major feature.
